Splash Page

Technical Description

This splash page serves as an introduction to the Ocean Realm website, displaying a visually appealing background, an animated loading effect, and an automatic redirection to the main content (home.html) after 4 seconds. The design focuses on user engagement while ensuring smooth navigation. The meta tag with http-equiv="refresh" is used to automatically redirect users to home.html after 4 seconds, ensuring a seamless transition. The structure of the page is built around several key div elements. The .image-container acts as the background, ensuring a full-screen ocean-themed image, while .left contains the main content, using a semi-transparent overlay to improve text readability. A fixed-position .logo-top-left div displays the project’s branding, making sure it remains visible regardless of scrolling. The .loading section features animated dots, creating a visual cue that the page is transitioning. Finally, the .page-editor-container provides a button linking to the Page Editor, allowing for content updates. From a styling perspective, the .image-container ensures that the background covers the entire screen using background-size: cover, preventing distortion. The .left section is styled with position: fixed, width: 50%, and a background-color: rgba(0, 0, 0, 0.5), providing a dark overlay that enhances text contrast. Typography is handled using the imported Poppins font, which gives the page a clean and modern appearance. Text elements (h1, h2, h3, p) are centrally aligned for an organized layout, with text-shadow applied to enhance readability. For user engagement, animations are incorporated. The @keyframes fadeIn effect makes the text appear gradually, while @keyframes jump animates the loading dots to give users a visual indication of transition. Interactive elements, such as the .page-editor-container a button, have a hover effect that changes background and text colors, improving user interaction.

Accessibility

Accessibility is a key consideration in the design of this splash page. To enhance contrast and readability, a semi-transparent black overlay (rgba(0, 0, 0, 0.5)) is used behind the text. This ensures that the white text remains highly readable against the background image. Additionally, text-shadow and high-contrast colors are applied to further improve visibility. The page is designed to be scalable and responsive, using vh, vw, and rem units instead of fixed pixel values. This allows elements to adjust dynamically across different screen sizes and devices. For keyboard navigation and focus, all interactive elements, such as the Page Editor button, are styled to provide clear visual feedback when hovered or focused. This ensures that users navigating with a keyboard or assistive technology can easily identify clickable elements. Finally, animations are kept subtle to prevent discomfort for users sensitive to motion. While the loading dots animate, they do so in a slow and controlled manner. The fade-in effects for text also ensure that content remains easy to read without abrupt transitions. By implementing these techniques, the splash page offers an engaging, accessible, and user-friendly experience while maintaining smooth functionality.

Volunteer Page

Technical Description

The volunteer page is structured using a combination of <html> and <css> techniques that enhance both functionality and design. The HTML document begins with the <!DOCTYPE html> declaration, which defines the document type and version of HTML being used. The <html> tag encompasses the entire content of the page, while the <head> section contains metadata, including the character set, viewport settings for responsive design, and links to external resources like stylesheets and icons. The <title> tag provides a concise description of the page, which is displayed in the browser tab. The main content of the page is organized within semantic HTML elements, such as <header>, <section>, and <footer>. The <header> contains the site logo and navigation menu, which is structured using an unordered list <ul>. Each navigation item is wrapped in a list item <li>, providing a clear and accessible way for users to navigate the site. The use of <div> elements helps to group related content, such as the volunteer cards and review sections, enhancing the layout and organization of the page. CSS is employed to style the page, utilizing properties like flexbox for layout management. The .card-container class uses display: flex to create a responsive layout for the volunteer cards, allowing them to adjust based on the screen size. The gap property provides spacing between the cards, while border-radius and box-shadow enhance the visual appeal by creating rounded corners and subtle depth. Background images for the volunteer cards are set using background-image, background-size, and background-position, ensuring that the images cover the card area effectively. Hover effects are implemented using the :hover pseudo-class, which changes the appearance of elements when the user hovers over them. For instance, the .volunteer-card:hover .card-info rule reveals additional information about the volunteer program, enhancing interactivity. The use of transitions, such as transition: right 0.3s ease-in-out, provides smooth animations, improving user experience.

Accessibility

Accessibility is a crucial aspect of web design, and several techniques have been implemented on this volunteer page to ensure it is usable for all individuals, including those with disabilities. The use of semantic HTML elements, such as <header>, <section>, and <footer>, helps screen readers understand the structure and hierarchy of the content, making navigation easier for users relying on assistive technologies. The navigation menu includes descriptive text within the <a> tags, allowing users to understand the purpose of each link clearly. Additionally, the page employs appropriate contrast ratios between text and background colors, ensuring readability for users with visual impairments. The buttons, such as those for submitting feedback, are clearly labeled with text, providing context for their function. The use of aria-labels or role attributes could further enhance accessibility, although they are not explicitly included in the provided code. The inclusion of alt text for images, such as the logo, ensures that users who cannot see the images can still understand their purpose. The layout is designed to be responsive, allowing users on various devices to access the content without difficulty. Overall, these accessibility considerations aim to create an inclusive experience for all users, promoting equal access to information and functionality on the volunteer page. .

Content Page

Technical Description

The "Careers in Marine Science" page is designed using a combination of HTML and CSS techniques that enhance both functionality and aesthetics. The document begins with the <!DOCTYPE html> declaration, which specifies the HTML version being used. The <html> tag wraps the entire content, while the <head> section includes essential metadata, such as the character set, viewport settings for responsive design, and links to external stylesheets and icons. The <title> tag provides a clear description of the page, which is displayed in the browser tab. The structure of the page is organized using semantic HTML elements, including <header>, <div>, and <footer>. The <header> contains the site logo and a navigation menu, structured with an unordered list (<ul>) for easy navigation. Each navigation item is wrapped in a list item (<li>), which improves accessibility and usability for users. The main content is encapsulated within a <div class="container">, which serves as a flexible container for the various sections of the page. CSS is utilized to style the page, employing properties like flexbox for layout management. The .container class uses display: flex and flex-direction: column to create a vertical layout, ensuring that the content is centered and well-organized. The background color is set to a semi-transparent black (rgba(0, 0, 0, 0.6)), which enhances readability against the white text. Headings are styled with distinct colors and sizes to create a clear hierarchy, using properties like color, font-size, and text-align. Images are included with responsive design in mind, using max-width: 50% to ensure they scale appropriately on different devices. The .quote class is styled with italic text and a light color to differentiate quotes from the main content, enhancing visual interest. The "back to top" button is designed with a fixed position, ensuring it remains visible as users scroll. It features a gradient background and hover effects that improve interactivity, utilizing CSS transitions for smooth animations.

Accessibility

Accessibility is a fundamental consideration in the design of the "Careers in Marine Science" page, ensuring that all users, including those with disabilities, can navigate and understand the content. The use of semantic HTML elements, such as <header>, <h1>, <h2>, and <footer>, provides a clear structure that assists screen readers in conveying the hierarchy and organization of the content. The navigation menu is built using an unordered list, which is a standard practice for creating accessible navigation. Color contrast is carefully considered, with white text on a dark background to ensure readability for users with visual impairments. The images include descriptive alt attributes, allowing users who rely on screen readers to understand the content and context of the images. The "back to top" button is clearly labeled with an upward arrow, providing a straightforward function for users to return to the top of the page. The page also employs responsive design principles, ensuring that content is accessible on various devices, including smartphones and tablets. This adaptability is crucial for users who may have different accessibility needs. Overall, these accessibility features aim to create an inclusive experience, allowing all users to engage with the information presented on the page effectively.

Challenges and Lessons Learned

During the implementation process of a webpage utilizing HTML and CSS, I encountered several challenges, particularly with positioning the divs and dealing with overlapping issues. Initially, I struggled to align the divs correctly, which resulted in a cluttered and unorganized layout. The overlapping elements made the webpage look chaotic and unprofessional. To resolve these issues, I explored various CSS properties such as position, float, and flexbox. By experimenting with these properties, I discovered that using flexbox provided a more flexible and efficient way to align and distribute space among the divs. Additionally, I learned to use z-index to manage the stacking order of overlapping elements, ensuring that each div appeared in the correct layer. Through this process, I gained a deeper understanding of CSS layout techniques and the importance of planning the structure of a webpage before diving into the code. This experience taught me the value of persistence and experimentation in problem-solving, and it highlighted the significance of continuous learning and adapting to new tools and methods in web development.
