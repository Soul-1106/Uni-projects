Home page

Technical Description

The homepage is structured using fundamental HTML elements, such as <header>, <nav>, <section>, and <div>, to create a clear and organized layout. The <header> contains the logo and navigation bar, with the logo positioned using absolute positioning to ensure it stays fixed at the top-left of the page. The <nav> holds an unordered list (<ul>) of navigation links that are styled for a clean horizontal layout. Each navigation link has a hover effect that provides additional information via a <div> with the class .hover-card, enhancing user interaction. The main content of the page is placed within a <section> with the id="home-page", which holds multiple content blocks represented by .content-container <div>s. These containers house an image, heading, and description, each styled to stand out against the background and draw attention to the page's key topics such as marine biodiversity and career opportunities in marine science.

The CSS uses several techniques to ensure the page is both functional and visually appealing. Flexbox is used in the navigation bar to space out links evenly, and positioning properties like position: absolute and position: relative are used for precise control over the layout of elements like the logo and content containers. For the content sections, a combination of background, backdrop-filter, and box-shadow creates visually engaging overlays and depth, while transform: scale(1.02) on hover adds interactivity to the .content-container elements. Media queries are applied to ensure responsiveness, making the layout adapt to various screen sizes. The use of CSS transitions and hover effects improves the user experience by adding smooth animations, such as enlarging images and displaying arrows on hover.

Accessibility

The homepage has been designed with accessibility in mind. Semantic HTML tags like <header>, <nav>, and <section> help screen readers interpret the page structure. Interactive elements are accessible via keyboard, and text links have clear and descriptive labels for easy navigation. The color contrast follows WCAG guidelines to ensure readability for users with visual impairments, and images are provided with alternative text descriptions for those relying on screen readers. These features enhance the page's inclusivity, offering a positive experience for all users.

Table Page

Technical Description

The table page is structured using fundamental HTML elements like <header>, <nav>, and <table> to create a clear, readable layout. The <header> contains the logo and navigation links, structured using an unordered list <ul>. The table itself is created using the <table>, <th>, and <td> tags to define rows and columns. Each row in the table represents an environmental issue with corresponding details and solutions. The <th> tags are used for the header row, while <td> tags define the content for each issue. To improve readability, the rows alternate colors using the nth-child CSS selector, providing a clear distinction between different entries. The <div> with the class .background-container wraps the table, applying additional styles for alignment and presentation.

CSS properties such as box-shadow, border-radius, and backdrop-filter are used to enhance the table's design, giving it a clean and modern appearance. Flexbox is used for layout management, ensuring the table remains centered on the page. The border-collapse property is applied to eliminate extra space between table cells, while border-bottom in the <td> tags ensures row separation. Hover effects are applied to table rows using the transform: scale(1.02) property, slightly enlarging them on hover for a more interactive user experience. The table's responsiveness is achieved through media queries, ensuring the page adjusts properly to various screen sizes.

Accessibility

The table page uses semantic HTML elements like <header>, <nav>, and <table> to ensure a clear structure. Interactive elements like navigation links are easy to navigate, with descriptions provided in <div class="hover-card"> for screen readers. The table uses <th> and <td> for better data readability, and all images have alt attributes for accessibility. Color contrast and responsive design are implemented to improve accessibility for users with visual impairments and on different screen sizes.

Additionally, the page supports keyboard navigation and ensures that all interactive elements are usable without a mouse, offering inclusivity for users relying on assistive technologies.

Content Page

Technical Description

This webpage utilizes semantic HTML elements like <header>, <nav>, <section>, and <footer>, which not only provide a clean and organized structure but also enhance SEO and accessibility. The navigation bar is composed of an unordered list (<ul>) with anchor links (<a>) for easy navigation, and each menu item includes a hover effect for user interaction. All images include descriptive <alt> text to improve accessibility for screen readers. Flexbox layout ensures responsiveness, adjusting content for various screen sizes. A viewport meta tag is used for mobile optimization, and clear headings and lists improve readability. The color contrast and design have been considered for better visual accessibility.

The content is organized into clear sections, and lists are formatted using <ul> and <li> elements to enhance readability. The design also ensures sufficient color contrast and responsive layouts for accessibility on all devices.

Interactive elements, including links and images, are easily navigable with a keyboard, offering inclusivity for users relying on assistive technologies. The footer provides relevant links with clear descriptions for additional navigation.

Challenges and Lessons Learned

Building this website presented challenges in responsive design, cross-browser compatibility, and performance optimization. Ensuring the site worked smoothly across different screen sizes required extensive use of CSS media queries and flexbox. Cross-browser issues were resolved by using widely supported CSS properties and thorough testing. To improve performance, I compressed images and used lazy loading techniques. I also focused on accessibility by adding proper alt text and ensuring keyboard navigation. These challenges taught me the importance of designing an inclusive, fast, and user-friendly website.

Compliance

For compliance, I ensured the website adhered to accessibility standards, including proper alt text for images, semantic HTML, and keyboard navigability. I also followed best practices for data privacy by avoiding the collection of sensitive user information. The design respects copyright laws by using royalty-free images and adhering to licensing agreements. This approach helped create a responsible, user-friendly website that meets legal and accessibility requirements.
